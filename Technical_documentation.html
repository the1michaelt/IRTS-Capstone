Integrated Registration Tracking System Technical Documentation

About the Project

The Integrated Registration Tracking System (IRTS) would serve as the storage hub for the academic credentials of students in an educational institution. Students may access their own course and grade history and choose courses in which to enroll. Employees may access multiple data points about any student's academic progress (post-Admission) and eligibility for graduation.

My Process

The User Stories for this app were derived from my recollection of the functions that I used to use in registration systems as a Registrar’s office employee. The intention was to re-create the system and build accessibility features into the design.

The tools were a combination the technologies listed below, my prior usability testing experience, and education in graphic design. Designing the wireframes and ERD were the easiest parts to finish. While writing the math calculations in algebra was easy, writing them in Python as APIs proved far more difficult

My plan for the sprint was to devote the first three days to drafting the wireframes and ERD and sketching the math calculations to return the expected results.

The overall pace was the balance of quick ideas and slow coding. Some functions required assistance to apply what was taught in the class to more advanced applications of them to satisfy the User Story requirements.

The initial build leading to the presentation took the 15-day sprint. It took an additional 15 days to code the calculations to work accurately.

API Endpoints

Person objects

  • Get Current Credits
    http://127.0.0.1:8000/api/auth/get_current_credits//

    A logged-in student may view his/her own current data on the Transcript page.

  • Get Current Semester
    http://127.0.0.1:8000/api/auth/get_current_semester//

    A logged-in student may view his/her own current data on the Transcript page.

  • Get Current GPA
    http://127.0.0.1:8000/api/auth/get_current_gpa//

    A logged-in student may view his/her own current data on the Transcript page.

Courses

  • http://127.0.0.1:8000/api/courses/courses_available/

    Displays all courses of a logged-in student at their current semester level on the AvailableCourses page.

  • http://127.0.0.1:8000/api/courses/post_create_courses/

    A logged-in employee may create a course via the AddCourses page.

  • http://127.0.0.1:8000/api/courses/delete_courses//

    A logged-in employee may delete a course via the AddCourses page.

Student Courses

  • http://127.0.0.1:8000/api/student_courses/post_student_into_courses/

    Allows a logged-in student placement into a course via the “enroll” button on the AvailableCourses page.

  • http://127.0.0.1:8000/api/student_courses/admin_gets_studentcourses//

    Displays all courses of a logged-in employee for a student selected via the FindStudentCourses page.

  • http://127.0.0.1:8000/api/student_courses/get_transcript/

    Displays all courses of a logged-in student on the Transcript page.

  • http://127.0.0.1:8000/api/student_courses/get_scheduled_courses/

    Displays all courses into which a logged-in student has clicked “enroll” from the menu of options on the AvailableCourses page.

  • http://127.0.0.1:8000/api/student_courses/put_grade_course_object//

    A logged-in employee may apply a grade to a course via the GradeCourse page.

  • http://127.0.0.1:8000/api/student_courses/delete_courses//

    A logged-in employee may delete a course via the CourseCatalog page.

    A logged-in student may delete a course via the “disenroll” button on the ScheduledCourses page.
